# 機能仕様書: 自然言語 OCI コストクエリ

**機能ブランチ**: `001-nl-cost-query`
**作成日**: 2026-03-17
**ステータス**: ドラフト
**入力**: ユーザー説明: "OCI のコスト管理コンソールはデータを提供するが、「先月と比べてどうか」「どのサービスが一番高いか」といった問いに即座に答えるには、UIでのフィルタ操作やエクスポートが必要。このエージェントは自然言語で期間やスコープを指定するだけで、サービス種別ごとのコスト内訳を返し、傾向の要約も行う。"

## ユーザーシナリオ & テスト *(必須)*

### ユーザーストーリー 1 - 期間別サービスコスト内訳 (優先度: P1)

クラウド運用エンジニアが、特定の期間における各 OCI サービスの
コストを知りたいと考えています。「先月のサービス別コストを教えて」
や「Show me cost breakdown for March 2026」のような自然言語
クエリを入力すると、システムはクエリを解釈し、OCI からコスト
データを取得し、コストの高い順にサービスごとの構造化された
内訳を返します。

**この優先度の理由**: これはコアバリュープロポジション ——
手動の OCI コンソールフィルタリングを単一の自然言語クエリで
置き換えること。これがなければ、エージェントの存在意義がありません。

**独立テスト**: 自然言語クエリに期間を指定して発行し、OCI コスト
管理コンソールのデータと一致する正確なサービス別コスト内訳が
返されることを検証することで、完全にテスト可能です。

**受入シナリオ**:

1. **前提** ユーザーが有効な OCI 認証情報を設定済みの場合、
   **操作** 「先月のサービス別コストを見せて」と尋ねると、
   **結果** システムは前月の各サービスのコストを含む表を
   コスト降順でソートし、合計行を末尾に付けて返します。

2. **前提** ユーザーが有効な OCI 認証情報を設定済みの場合、
   **操作** 「Show costs for January 2026」と尋ねると、
   **結果** システムは 2026年1月1日〜31日のサービス別コスト
   内訳を、OCI で設定された通貨で返します。

3. **前提** ユーザーがコストデータのない期間を指定した場合、
   **操作** クエリが処理されると、
   **結果** システムは指定された期間にコストデータがない旨の
   明確なメッセージを返します。

4. **前提** ユーザーのクエリが期間について曖昧な場合、
   **操作** システムが意図した期間を確信をもって判断できない場合、
   **結果** 選択肢を提示した確認質問を返します
   （例: 「先月とは暦月ですか？直近30日間ですか？」）。

---

### ユーザーストーリー 2 - コスト傾向比較 (優先度: P2)

クラウド運用エンジニアが、コストが時間とともにどう変化したかを
理解したいと考えています。「先月と比べてコストはどうなった？」
や「Compare costs between February and March」のような質問を
します。システムはサービスごとの差分、変化率、および傾向の
簡潔な自然言語サマリーを含む比較結果を返します。

**この優先度の理由**: 傾向分析は機能説明で最も多く言及された
2番目のユースケース。P1 のコスト内訳に比較の次元を追加します。

**独立テスト**: 比較クエリを発行し、サービスごとの差分、
変化率、および人間が読めるサマリーが出力に含まれることを
検証することでテスト可能です。

**受入シナリオ**:

1. **前提** 2つの期間のコストデータが存在する場合、
   **操作** ユーザーが「先月と今月を比較して」と尋ねると、
   **結果** システムはサービス名、前期間コスト、当期間コスト、
   絶対変化額、変化率のカラムを持つ表を、絶対変化額の
   降順でソートして返します。

2. **前提** 2つの期間のコストデータが存在する場合、
   **操作** 比較が表示されると、
   **結果** システムは以下を強調する自然言語サマリーも提供します:
   コスト総額の変化、増加額が最大のトップ3サービス、
   および顕著に減少したサービス。

3. **前提** 比較対象の2期間のうち1つにデータがない場合、
   **操作** クエリが処理されると、
   **結果** システムはどの期間にデータがないかを説明するエラーを
   返し、利用可能な日付範囲を提案します。

---

### ユーザーストーリー 3 - スコープ指定コストクエリ (優先度: P3)

クラウド運用エンジニアが、OCI 内の特定のコンパートメント、タグ、
またはサービスタイプにコストクエリを絞り込みたいと考えています。
「Compute のコストだけ見せて」や「Show costs for the production
compartment last quarter」のような質問をします。システムは指定
されたスコープでコストデータをフィルタリングしてから結果を返します。

**この優先度の理由**: スコープ指定クエリは精度を追加しますが、
コア MVP には不要です。最初の2つのストーリーが既に主要な
価値を提供します。

**独立テスト**: スコープ指定クエリを発行し、指定されたフィルタ
（サービスタイプ、コンパートメント、またはタグ）に一致する
コストのみが結果に含まれることを検証することでテスト可能です。

**受入シナリオ**:

1. **前提** 複数サービスのコストデータが存在する場合、
   **操作** ユーザーが「Compute の先月のコストは？」と尋ねると、
   **結果** システムは前月の Compute 関連コスト明細のみを返します。

2. **前提** コストデータがコンパートメント別にタグ付けされている場合、
   **操作** ユーザーが「Show production compartment costs
   for last month」と尋ねると、
   **結果** システムは指定されたコンパートメントのみに
   フィルタリングされたコストを返します。

3. **前提** ユーザーが存在しないスコープを指定した場合、
   **操作** クエリが処理されると、
   **結果** システムは一致するスコープが見つからなかった旨の
   メッセージを返し、類似する利用可能なスコープを提案します。

---

### エッジケース

- OCI 認証情報が期限切れまたは無効な場合:
  システムは認証情報の再設定方法のガイダンスを含む明確な
  認証エラーを返さなければなりません (MUST)。
- OCI コスト管理 API が一時的に利用不可の場合:
  システムは具体的な障害理由を含むリトライ可能なエラー
  メッセージを返さなければなりません (MUST)。
- ユーザーの自然言語クエリがコストクエリとして認識できない場合:
  システムはユーザーが試せるクエリ例を返さなければ
  なりません (MUST)。
- コストデータが非常に大きい場合（数千の明細）:
  システムはデフォルトでサービスレベルに集約し、必要に応じて
  ドリルダウンする手段を提供しなければなりません (MUST)。
- ユーザーが日本語または英語以外の言語でクエリした場合:
  システムはクエリと同じ言語で応答しなければなりません
  （日本語と英語をサポート。他の言語にはサポート対象言語を
  示すメッセージを返します）(MUST)。

## 要件 *(必須)*

### 機能要件

- **FR-001**: システムは日本語と英語の両方で自然言語クエリを
  受け付け、コストクエリパラメータ（期間、スコープ、比較対象）を
  指定できなければなりません (MUST)。
- **FR-002**: システムは自然言語入力から構造化されたクエリ
  パラメータ（開始日、終了日、サービスフィルタ、コンパートメント
  フィルタ、タグフィルタ）を抽出しなければなりません (MUST)。
- **FR-003**: システムは抽出されたパラメータを使用して OCI コスト
  管理サービスからコストデータを取得しなければなりません (MUST)。
- **FR-004**: システムはコスト降順でソートされたサービス別コスト
  内訳を合計行付きで返さなければなりません (MUST)。
- **FR-005**: システムは相対的な時間参照（「last month」、「先月」、
  「past 90 days」、「this quarter」）をサポートし、絶対的な
  日付範囲に解決しなければなりません (MUST)。
- **FR-006**: システムは2期間を比較する際に、トップの増加・減少・
  全体的な変化を強調する自然言語トレンドサマリーを生成
  しなければなりません (MUST)。
- **FR-007**: システムは OCI コンパートメント名または OCID による
  フィルタリングをサポートしなければなりません (MUST)。
- **FR-008**: システムは OCI サービスタイプ（例: Compute、
  Object Storage、Networking）によるフィルタリングをサポート
  しなければなりません (MUST)。
- **FR-009**: システムは OCI テナンシーに関連付けられた通貨を
  使用して、通貨値を一貫してフォーマットしなければ
  なりません (MUST)。
- **FR-010**: システムはクエリが曖昧な場合、誤って推測する
  のではなく確認質問をしなければなりません (MUST)。
- **FR-011**: システムはユーザーの入力がコスト関連の質問として
  解釈できない場合、クエリ例を提供しなければなりません (MUST)。
- **FR-012**: システムは OCI API エラーを適切に処理し、
  ユーザーフレンドリーなエラーメッセージを返さなければ
  なりません (MUST)。

### 主要エンティティ

- **CostQuery（コストクエリ）**: パースされたユーザーリクエストを
  表す — 期間（開始/終了日）、スコープフィルタ（コンパートメント、
  サービスタイプ、タグ）、クエリタイプ（内訳または比較）、
  比較対象期間（該当する場合）を含む。
- **CostLineItem（コスト明細）**: OCI からの単一のコストエントリ
  — サービス名、リソースタイプ、コンパートメント、コスト金額、
  通貨、利用期間を含む。
- **CostBreakdown（コスト内訳）**: サービス別にグループ化された
  コスト明細の集約ビュー — サービス名、合計コスト、
  全体に占める割合、順位を含む。
- **CostComparison（コスト比較）**: 2つのコスト内訳のペア分析
  — サービスごとの差分（絶対値と割合）、全体の変化、
  傾向の方向を含む。
- **TrendSummary（傾向サマリー）**: コスト比較から導出された
  自然言語ナラティブ — 注目すべき変化、異常値、
  全体的な軌跡を強調。

### 前提条件

- ユーザーはこのツールを使用する前に OCI 認証情報（API キー
  またはインスタンスプリンシパル）を設定済みであること。
  認証情報のセットアップはこの機能のスコープ外です。
- OCI テナンシーでコスト管理が有効化されており、ユーザーが
  コストデータを読み取るために必要な IAM 権限を持っていること。
- コストデータの粒度は日次。時間単位の内訳は初期バージョンでは
  サポートされません。
- システムは CLI エージェント（テキスト入出力）として動作します。
  Web UI はこの機能のスコープ外です。
  ※ デプロイ用の HTTP API ラッパーはスコープ内です。
- 通貨は OCI テナンシーの設定によって決定されます。
  マルチ通貨サポートは不要です。

## 成功基準 *(必須)*

### 測定可能な成果

- **SC-001**: ユーザーは自然言語の一文を入力するだけで、
  メニューやフィルタを操作することなく、任意の指定期間の
  サービス別コスト内訳を取得できること。
- **SC-002**: 日本語または英語の適切な自然言語クエリの 90% が、
  確認を必要とせず初回で正しく解釈されること。
- **SC-003**: 最大3ヶ月のデータを対象とするクエリに対し、
  コスト内訳結果が5秒以内に返されること。
- **SC-004**: 傾向比較サマリーが、同じデータの手動分析と比較して
  95% の精度でコスト増加トップ3サービスを正しく特定すること。
- **SC-005**: ユーザーがクエリ入力からコストの回答を理解するまでの
  時間が30秒未満であること（OCI コンソール経由での推定
  3〜5分と比較）。
- **SC-006**: システムは認識できない入力の 100% に対して有用な
  応答（エラーメッセージ、確認質問、またはクエリ例）を提供し、
  空の応答や不明瞭な応答を返さないこと。
